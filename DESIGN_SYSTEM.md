# Astrul: Design System
**Team Astrul: S³C Design Competition 2026**

---

## 1. Visual Identity
Our design is based on the idea of "Functional Calm." We use a very limited color palette to keep the focus on student needs and tasks.

### Color Palette
- **Navy (Primary)**: #001F3F. Used for headers, primary buttons, and branding. It represents NUST's institutional strength.
- **Paper (Background)**: #F9FAFB. A soft white that reduces eye strain and feels like a clean slate.
- **Mist (Surface)**: #F3F4F6. Used for secondary cards and background sections.
- **Ink (Text)**: #111827. A deep charcoal for high readability.

### Status Colors (Functional Only)
We only use bright colors to show status. This prevents the user from getting distracted.
- **Green**: Resolved / Success.
- **Yellow**: In Progress / High Priority.
- **Red**: Urgent / Action Needed.
- **Blue**: Submitted / Routed.

## 2. Typography
We use **Geist Sans** for its clarity and modern look.
- **Headings**: Semi-bold, tight tracking. Used for page titles and section headers.
- **Body Text**: 14px default. This ensures readability on all devices.
- **Monospace**: Used exclusively for Request IDs to make them look distinct and easy to copy.

## 3. UI Components
- **Card**: Soft borders (8px) with a subtle shadow. Cards group related information together without feeling heavy.
- **Button**: Large touch targets (44px minimum). Primary buttons are solid navy. Secondary buttons are white with a clean border.
- **SLA Ring**: A custom circular progress bar. It uses status colors to show how much time is left for a request.

## 4. Design Principles
1. **Consistency First**: The same icons and colors mean the same thing everywhere in the app.
2. **Clear Hierarchy**: The most important information (like the request status) is always at the top.
3. **No Decoration**: If an element does not help the user complete a task, we remove it.
4. **Mobile First**: Every component is designed to work perfectly with a thumb on a phone screen.
