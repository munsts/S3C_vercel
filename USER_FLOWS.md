# Astrul: User Flows
**Team Astrul: S³C Design Competition 2026**

---

## 1. Student Submission Flow
This is the core flow for a student needing help.
1. **Entry**: Student lands on the Home Dashboard.
2. **Action**: Taps the "New Request" button.
3. **Category Selection**: Student selects a category (like Finance or Hostel).
4. **Description**: Student types the issue and can add details.
5. **Review**: A quick review screen shows the summary.
6. **Confirmation**: Student submits and sees a success message with an ID.

## 2. Staff Triage and Chat Flow
This is how staff members manage incoming work.
1. **Queue View**: Staff opens the Queue and sees all active requests.
2. **Selection**: Staff taps a request title to open the details.
3. **Engagement**: Staff reads the description and taps "Triage" to mark it as In Progress.
4. **Conversation**: Staff types a reply to the student in the chat box.
5. **Resolution**: Once fixed, staff taps "Resolve" to close the request.

## 3. Real-Time Tracking Flow
This is how a student stays updated.
1. **Alert**: Student receives a notification about a status change or a new message.
2. **Navigation**: Student taps the notification to go directly to the request page.
3. **Information**: Student sees the updated SLA ring and staff response.
4. **Follow-up**: Student can reply back in the chat if they have more questions.

## 4. Academic Check Flow
This is the "All-in-One" surface benefit.
1. **Home**: Student opens the app.
2. **Quick View**: Student sees their current GPA and next class immediately.
3. **Deadlines**: Student scrolls down to see LMS assignments due today.
4. **Action**: Student can tap an assignment to go to the LMS or return to support services.
